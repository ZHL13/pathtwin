# PathTwin v0.1.3

Patch release focused on progress visibility, safer remembered selections, and release packaging cleanup.

## Fixed

- Restored last-session folder selections without turning an unloaded parent folder into a full selection.
- Kept the settings Remote Root picker lightweight for large folders by listing only one level at a time.
- Excluded `.pt` metadata folders from directory listing, file scanning, skeleton creation, and sync planning.
- Updated default log/history paths to `<LocalRoot>.pt\logs` and `<LocalRoot>.pt\history`.

## Improved

- Added real-time progress details for Start Work Session and End Work Session.
- Moved directory skeleton creation off the UI thread.
- Skipped empty unselected skeleton folders during push planning.
- Cleaned README and release checklist wording for the public release workflow.

## Included Artifacts

- `PathTwin-0.1.3-win-x64.exe`: versioned standalone executable.
- `PathTwin-latest-win-x64.exe`: same standalone executable under a stable latest filename.

GitHub also provides automatic source archives for the release tag. No separate source zip is uploaded.

## Notes

The public release does not include `rclone.exe`. Users who want rclone support should download it from the official rclone downloads page and set the executable path in Settings: https://rclone.org/downloads/
