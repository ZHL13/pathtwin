# PathTwin v0.1.5

Start Work safety release for unfinished-session detection and stale local cache cleanup.

## Added

- Previous-session status check before Start Work cleanup.
- `Status` in session JSON and manifest JSON, with old session files inferred from `EndedAt`.
- Local trash location for Start Work cleanup under `<localRoot>.pt\trash\<session>_StartWorkCleanup`.
- `MoveCleanedContentToLocalTrash` profile setting, defaulting to `true`.

## Changed

- Start Work now stops before local cleanup if the latest previous session is `Active`, `Failed`, or `Interrupted`.
- Start Work moves unselected stale local content out of the working cache before recreating the skeleton and pulling selected folders.
- Selected folders and required ancestor directories are preserved during cleanup.
- Cleanup planning validates relative paths, preserves app metadata folders, and avoids following reparse points during enumeration.
- Edit Profile no longer drops an active session back to folder selection after Cancel.
- Saving session-affecting profile changes during an active session now ends the old session with the original settings before saving the new profile and returning to folder selection.
- Sync progress detail uses a fixed single-line area with ellipsis so the activity log stays visually stable.

## Included Artifacts

- `PathTwin-0.1.5-win-x64.exe`: versioned standalone executable.
- `PathTwin-latest-win-x64.exe`: same standalone executable under a stable latest filename.

GitHub also provides automatic source archives for the release tag. No separate source zip is uploaded.

## Notes

The public release does not include `rclone.exe`. Users who want rclone support should download it from the official rclone downloads page and set the executable path in Settings: https://rclone.org/downloads/
