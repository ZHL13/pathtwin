# PathTwin v0.1.4

Workflow refinement release for shallow local skeletons, append-only folder additions during a session, and categorized final push behavior.

## Added

- `SkeletonDepth` profile setting, defaulting to `2`, for shallow local directory skeleton creation.
- Add Folder mode during an active session.
- Resume Sync for newly added folders only.
- Append-only session folder selection: folders already pulled in the current session are shown checked and locked.
- Session manifest fields for skeleton depth, initial selections, added selections, selected paths, session events, and pull/push log paths.

## Changed

- Start Work creates the configured shallow skeleton, then fully pulls only manually selected folders while preserving relative paths.
- End Work now separates selected folders, unselected non-empty folders, and empty skeleton folders.
- Selected folders are mirror-pushed with existing remote backup behavior.
- Unselected non-empty folders are update-pushed with copy/update behavior and no mirror deletes.
- Empty skeleton folders are ignored and summarized by count in the push log.
- rclone copy no longer creates empty source directories when used for unselected update-only folders.

## UI and Docs

- Added README logo and application screenshots.
- Fixed top-right header buttons so normal, hover, pressed, and disabled states remain readable on the dark header.

## Included Artifacts

- `PathTwin-0.1.4-win-x64.exe`: versioned standalone executable.
- `PathTwin-latest-win-x64.exe`: same standalone executable under a stable latest filename.

GitHub also provides automatic source archives for the release tag. No separate source zip is uploaded.

## Notes

The public release does not include `rclone.exe`. Users who want rclone support should download it from the official rclone downloads page and set the executable path in Settings: https://rclone.org/downloads/
