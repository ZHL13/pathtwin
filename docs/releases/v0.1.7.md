# PathTwin v0.1.7

Performance and sync-status refinement release.

## Changed

- End Work scans selected local and remote folders concurrently. As soon as both states for a file are available and its three-way comparison confirms a change, PathTwin queues that file for transfer.
- Confirmed file operations and folder transfers use a single queue so they do not compete for the same network bandwidth. A later conflict stops newly queued work; changes that already started are retained and reported.
- Pull, selected mirror push, and update-only push skip nested duplicate roots before entering the transfer queue.
- Each concurrent folder task gets a separate log file; the main session log records the task lifecycle.
- During synchronization, the header identifies the active backend to the left of the `Logs` button, and separates real-time file comparison from file changes in two scrolling columns.
- Profiles can now enable or disable rclone explicitly. When enabled with a valid executable, rclone scans remote files through `lsjson` and handles transfers.
- Added comparison modes: `Hybrid` (default), `Fast`, and `Content`.
- Startup now identifies unfinished prior sessions with the saved failure phase, error summary, last recorded activity, and an action to open the relevant logs. Normal Start stays protected; `Ignore Previous Session and Force Sync` is an explicit logged override.
- Native file-system synchronization now reports progress while checking large folder trees, Hybrid hashes, and mirror cleanup after comparison. Closing PathTwin now requires a state-aware confirmation, with an explicit force-exit warning for active syncs and sessions.
- Native Start and Resume pulls now stream candidate file operations into the single transfer worker during remote scanning. With rclone active, `Fast` uses metadata; `Hybrid` lists both sides through rclone and checksum-verifies only metadata-mismatched paths before conditional copy; and `Content` uses full rclone checksums, including final selected-folder reconciliation.
- rclone Hybrid now streams checksum status and throttled file events to the UI. Its empty-directory creation reports a separate phase and creates local or SMB destination directories directly, avoiding one rclone process per directory.
- The main window renders before the local profile configuration is loaded. `Edit Profile` becomes available after that load completes.
