# PathTwin v0.1.6

Cleanup refinement release for Start Work local-cache handling.

## Changed

- Start Work cleanup now sends stale unselected local cache items to the Windows Recycle Bin.
- Cleanup no longer writes removed items into `.pt\trash`.
- Cleanup now runs after selected remote files are scanned and the shallow remote skeleton is compared.
- Existing local skeleton directories that still belong to the current remote skeleton are preserved, so PathTwin avoids deleting a directory only to recreate it moments later.
- The active profile model no longer exposes the local-trash cleanup setting. Older config files containing that field remain readable.

## Included Artifacts

- `PathTwin-0.1.6-win-x64.exe`: versioned standalone executable.
- `PathTwin-latest-win-x64.exe`: same standalone executable under a stable latest filename.

GitHub also provides automatic source archives for the release tag. No separate source zip is uploaded.

## Notes

The public release does not include `rclone.exe`. Users who want rclone support should download it from the official rclone downloads page and set the executable path in Settings: https://rclone.org/downloads/
